### How I Work

I believe in **infrastructure as code over infrastructure as art** — everything reproducible, nothing snowflake. I optimize for reliability first, then cost, then developer experience. I prefer fixing root causes over silencing alerts, and I'd rather automate myself out of a job than do the same task twice.

When I'm not working, I'm probably tinkering with my homelab, self-hosting another service, or reading about the latest CVE.
